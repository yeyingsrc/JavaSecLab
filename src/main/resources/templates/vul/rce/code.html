RCE - 远程代码执行 
  RCE(Remote Code Execution)：攻击者让服务端执行非预期的系统命令或代码逻辑。常见类型包括命令注入(Command Injection)和代码注入(Code Injection)。
  代码注入通常发生在业务把不可信输入交给脚本引擎、表达式引擎、模板引擎或动态编译执行。风险不只来自“执行命令”，读取文件、访问网络、反射调用和修改业务数据同样危险。

漏洞场景：Groovy代码注入 流量分析

tips
问题点：
  1、GroovyShell.evaluate 会把字符串当作 Groovy 代码执行，不可信输入进入这里就是高危代码注入。
  2、Groovy 能调用 Java API、反射、文件、网络和进程执行能力，危害不局限于弹计算器。
  3、黑名单过滤 execute、Runtime 等关键字很容易被编码、反射、字符串拼接或其他 API 绕过。

安全编码建议：
  1、不要把用户输入交给 GroovyShell、ScriptEngine、模板引擎或表达式引擎直接执行。
  2、把“可执行脚本”改造成“可选择动作”：用户只能选择动作名，服务端执行固定 Java 逻辑。
  3、如果确实需要脚本能力，应使用隔离执行环境、最小权限、资源限制、审计和人工发布流程。
  4、脚本文本白名单只适合内部发布的固定脚本，不适合让外部用户提交任意脚本。
