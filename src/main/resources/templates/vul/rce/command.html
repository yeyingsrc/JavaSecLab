RCE - 远程代码执行 
  RCE(Remote Code Execution)：攻击者让服务端执行非预期的系统命令或代码逻辑。常见类型包括命令注入(Command Injection)和代码注入(Code Injection)。
  命令注入的本质是：不可信输入进入系统命令或 shell 语法结构，攻击者可以改变原命令语义。代码注入的本质是：不可信输入被解释器、表达式引擎或脚本引擎当作代码执行。

漏洞场景：ProcessBuilder 流量分析

tips
问题点：
  1、ProcessBuilder 本身不是漏洞，危险点在于把用户输入放进 sh -c 这类 shell 解释上下文。
  2、shell 会解析 ;、&&、|、$()、反引号、重定向等元字符，攻击者可拼接额外命令。
  3、如果业务必须调用系统命令，应避免 shell，改用固定参数数组并限制可选动作。

tips

                                        

                                            
审计关注点：
  1、Runtime.exec() 可以直接执行用户可控命令，尤其要关注从请求参数、Header、配置、数据库进入的命令字符串。
  2、Runtime.exec(String) 不等同于 shell，但仍允许攻击者控制要执行的程序和参数。
  3、若代码使用 /bin/sh -c、cmd.exe /c 包装 Runtime.exec()，风险会升级为完整 shell 命令注入。

tips

                                        

                                            
安全编码建议：
  1、首选业务 API 或语言内置能力，能不用系统命令就不要调用系统命令。
  2、必须执行命令时，让用户只选择业务动作，再由服务端映射到固定命令和固定参数。
  3、不要把用户输入交给 sh -c、cmd.exe /c，也不要拼接完整命令字符串。
  4、设置最小权限、工作目录、超时、输出长度限制和审计日志，避免命令卡死或扩大影响面。
