任意文件操作 - 文件读取 

  任意文件读取：应用将用户可控的文件名或路径直接用于文件读取，未做目录边界限制和路径标准化，攻击者可通过绝对路径或 ../ 目录穿越读取敏感文件，造成配置、密钥、源码或系统信息泄露。

漏洞场景：原生漏洞场景 流量分析

文件路径没做限制，可使用绝对路径或 ../ 遍历读取文件

安全场景：文件读取目录限制

安全场景将读取范围限制在上传目录内，可尝试读取目录内文件或使用 /etc/hosts 验证拦截效果

tips
安全编码建议：
  1、先确定允许访问的根目录，再使用 resolve + normalize 标准化目标路径。
  2、使用 startsWith(basePath) 校验目录边界，避免只做字符串替换或黑名单过滤。
  3、对已存在文件再校验 toRealPath，避免通过符号链接绕出允许目录。
  4、按业务使用文件 ID、枚举名或白名单映射，尽量不让用户直接传入物理路径。
  5、只读取普通文件，并结合权限控制、审计日志和敏感文件隔离。
