任意文件操作 - 文件删除 

  任意文件删除：应用将用户可控路径直接用于删除操作，未限制目录边界和文件类型，攻击者可能通过绝对路径或 ../ 目录穿越删除业务文件、上传文件、日志或配置，导致数据丢失、拒绝服务和审计证据破坏。

漏洞场景：路径未限制 流量分析

文件路径没做限制，可使用../遍历删除任意文件

tips
测试提醒：
  文件删除具有破坏性，真实测试应只使用临时文件，并提前确认路径、权限和恢复方案。

安全场景：文件删除目录限制

安全场景将删除范围限制在上传目录内，可使用 ../test 验证拦截效果

tips
安全编码建议：
  1、删除接口优先接收文件 ID，由后端查询文件归属、状态和真实路径。
  2、标准化路径后校验目录边界，避免 canonicalPath 字符串前缀误判。
  3、对已存在文件再校验 toRealPath，避免通过符号链接绕出允许目录。
  4、只允许删除普通文件，不允许删除目录或系统关键路径。
  5、删除前做业务授权、二次确认或回收站机制，并记录审计日志。
