SPEL - 表达式注入 

  SpEL(Spring Expression Language) 是 Spring 提供的表达式语言，可在运行时访问属性、调用方法、进行类型引用和表达式计算。若用户输入被直接传给 SpelExpressionParser.parseExpression() 并在 StandardEvaluationContext 中执行，表达式就可能从普通动态计算变成任意方法调用甚至命令执行。
  修复重点不是简单过滤关键字，而是避免解析不可信表达式；确需支持表达式时，应使用 SimpleEvaluationContext、白名单能力模型、固定表达式模板和业务参数绑定，禁止 Java 类型引用、构造函数、Bean 引用和任意方法调用。

tips
  典型审计链路是“不可信输入 -> SpelExpressionParser.parseExpression() -> Expression.getValue()”。当上下文使用 StandardEvaluationContext 时，表达式可访问 T() 类型引用、静态方法、构造函数和对象方法，危险表达式可能执行系统命令。审计时重点关注表达式内容是否完全由用户控制，以及上下文是否限制能力。
