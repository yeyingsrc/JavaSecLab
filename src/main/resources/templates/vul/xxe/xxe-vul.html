XXE (XML External Entity Injection) 
  XXE(XML外部实体注入)：应用解析不可信 XML 时允许 DTD 或外部实体，攻击者可通过 SYSTEM/PUBLIC 外部实体让解析器读取本地文件、访问内网地址、触发 SSRF，或利用实体膨胀造成拒绝服务。
  常见入口：Content-Type 为 text/xml/application/xml 的接口、SOAP、SAML、SVG、Office/Excel 预览、XML 配置导入、文件解析插件、消息队列或第三方回调中的 XML 数据。

漏洞场景：XMLReader

XMLReader 未禁用 DTD 和外部实体时，可能解析攻击者提供的外部实体

漏洞场景：SAXParser

SAXParser 解析不可信 XML 时同样需要显式关闭 DTD 和外部实体

tips
  不同 JDK、解析器实现和库版本的默认安全特性并不完全一致。审计时不要依赖“默认安全”，应在创建解析器后显式禁用 DOCTYPE、外部通用实体、外部参数实体和外部 DTD 加载。

漏洞场景：DocumentBuilder

DocumentBuilder 解析 DOM 文档时如果未禁用外部实体，同样会展开 SYSTEM/PUBLIC 实体

tips
  DOM、SAX、StAX、JAXB 等 XML 解析入口都要单独确认安全配置；不要只修某一个解析器封装。
